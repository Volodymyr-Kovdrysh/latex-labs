# ЛАБОРАТОРНА РОБОТА №1

## Створення та компіляція першого LaTeX-документа

---

## Мета роботи

Навчитися:

* встановлювати LaTeX-середовище;
* компілювати документ;
* аналізувати помилки;
* створювати структурований документ;
* оформлювати відповіді у вигляді повноцінного PDF-файлу.

---

## Формат виконання

Створити **один файл `lab1.tex`**, у якому:

* виконати всі завдання;
* дати письмові розгорнуті відповіді на запитання;
* оформити структуру документа;
* додати пояснення до кожного виконаного пункту.

У Google Classroom завантажується **тільки PDF-файл**.
Файли `.tex`, `.log`, `.aux` не надсилаються.

---

# ПРАКТИЧНА ЧАСТИНА

---

## Завдання 1. Налаштування середовища

1. Встановити TeX Live / MiKTeX або використати Overleaf.
2. Перевірити наявність компілятора командою:

```
pdflatex --version
```

---

### У документі оформити розділ:

```latex
\section{Середовище роботи}
```

У цьому розділі необхідно:

1. Вказати операційну систему.
2. Вказати встановлений дистрибутив LaTeX.
3. Вказати редактор.
4. Навести версію `pdflatex`.
5. Коротко пояснити, що таке компілятор у контексті LaTeX.

Відповіді повинні бути оформлені у вигляді зв’язного тексту (не просто список із 4 слів).

---

## Завдання 2. Створення структури документа

Створити базову структуру:

* титульна інформація;
* зміст;
* щонайменше 3 розділи;
* щонайменше 2 підрозділи.

---

### У документі обов’язково повинні бути:

```latex
\title{}
\author{}
\date{}
\maketitle
\tableofcontents
```

Після першої компіляції виконати повторну компіляцію.

---

### У розділі:

```latex
\section{Структура документа}
```

Пояснити:

1. Що таке преамбула документа?
2. Яка команда визначає клас документа?
3. Для чого використовується `\tableofcontents`?
4. Чому зміст формується лише після повторної компіляції?
5. Чим відрізняється `\section` від `\subsection`?

Пояснення повинні бути власними словами.

---

## Завдання 3. Аналіз лог-файлу

Після компіляції відкрити файл `.log`.

---

### У розділі:

```latex
\section{Аналіз лог-файлу}
```

Необхідно:

1. Вказати, який компілятор використано.
2. Знайти та перелічити підключені пакети.
3. Вказати, чи є попередження (Warning).
4. Пояснити повідомлення:

```
LaTeX Warning: Label(s) may have changed.
```

5. Пояснити, чим відрізняється Warning від Error.

---

## Завдання 4. Виклик та аналіз помилок

Навмисно створити помилки:

1. Видалити закриваючу дужку в команді `\section`.
2. Видалити `\end{document}`.

Спробувати скомпілювати документ.

---

### У розділі:

```latex
\section{Аналіз помилок}
```

Описати:

1. Яке повідомлення з’явилося?
2. Що означає "Emergency stop"?
3. Як знайти рядок з помилкою?
4. Чому LaTeX іноді вказує помилку не в тому місці, де вона виникла?
5. Чому важливо читати перше повідомлення про помилку, а не останнє?

---

## Завдання 5. Самостійна модифікація документа

Необхідно:

* змінити поля на 3 см;
* встановити фіксовану дату;
* додати розділ без нумерації;
* змінити розмір шрифту документа;
* увімкнути нумерацію сторінок.

---

### У розділі:

```latex
\section{Модифікація документа}
```

Пояснити:

1. Який пакет використовується для зміни полів?
2. Як створити розділ без номера?
3. Де задається розмір шрифту документа?
4. Яка команда керує нумерацією сторінок?
5. Чи можна змінити стиль нумерації (арабська / римська)?

---

## Висновки

У кінці документа обов’язково додати розділ:

```latex
\section*{Висновки}
```

У висновках описати:

* що нового було вивчено;
* які труднощі виникли;
* що виявилося найскладнішим.

---

# Вимоги до оформлення

* Документ повинен компілюватися без помилок.
* Повинна бути коректна структура.
* Зміст повинен відповідати розділам.
* Відповіді повинні бути розгорнутими.
* PDF має містити титульну сторінку.
